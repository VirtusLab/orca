# Review fix-up plan

Consolidated from 7 reviewer agents over the last 12 commits. Priority order;
each tier should typically be one commit (bundled when changes are tightly
coupled). Code review after each commit.

---

## Tier 1 — critical correctness + leaks

Bundle into one commit: all three share the lifecycle theme.

- [x] **T1. Port-bind NPE.** `AskUserMcpServer.start` constructs the
      `stopFn` closure with `() => binding.stop()` — if
      `NettySyncServer().start()` throws, `binding` is never assigned;
      `mcpServer.close()` later NPEs. Move binding inside try, or build
      the closure after start.

- [x] **T2. Drainer thread + bridge channel leak.** `ClaudeConversation.onFinalize`
      stops the MCP server but doesn't close the bridge. The daemon
      drainer parks on `bridge.take()` forever — until flow scope ends.
      Long flows accumulate one parked thread + closed-over conversation
      graph per interactive call. Add `AskUserBridge.close()`, call from
      `onFinalize` before `mcpServer.close()`. Also: track in-flight
      reply channels, signal them on close so the Netty worker returns.

- [x] **T3. Netty worker stranded mid-`ask`.** Same fix as T2 — closing
      the bridge errors any blocked `reply.receive()`. After T2's
      `bridge.close()`, the handler unwinds with a tool error, the agent
      sees a clean failure rather than the binding-stop force-killing
      the worker.

## Tier 2 — important

Each its own commit.

- [x] **T4. Enforce `canAskUser` at compile time.** Today
      `Plan.interactive.from(prompt, codex)` compiles and silently
      degrades. Add a typeclass `CanAskUser[B <: BackendTag]` with an
      instance for `ClaudeCode` only; constrain
      `Plan.interactive.from`'s `B` to require it. Keep the runtime
      `canAskUser` flag for programmatic queries.

- [x] **T5. `TerminalEventListener` lock + mailbox-full stall.** The
      lock is held across `output.log(...)` / `output.setStatus(...)`
      tells, which can block if the actor mailbox is full. Concurrent
      `currentIndent()` reads from the body thread then block on a
      listener stuck on a mailbox. Two-phase: mutate stage state + format
      under lock, release lock, then dispatch.

- [x] **T6. Bounded `StreamConversation.eventQueue`.** No cap +
      non-blocking `offer` = unbounded growth if consumer is slow.
      Switch to a bounded queue with blocking `put`; backpressure flows
      naturally to the subprocess via pipe buffers.

- [x] **T7. Delete `.orca-mcp-<port>.json` on conversation end.**
      Register an `AutoCloseable` in `sessionResources` that
      `os.remove`s the config file. Long flows currently accumulate
      orphans.
